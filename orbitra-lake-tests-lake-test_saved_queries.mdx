## Classes

### `TestSavedQueries`


Test suite for the saved queries feature.

Tests are organized in sections: CRUD, SQL validation, versioning, query, and metadata.
A shared table 'test_data' is created in setup for query tests.


**Methods:**

#### `test_can_create_saved_query_after_table_deletion`

```python
test_can_create_saved_query_after_table_deletion(self)
```

Test that a saved query can be created after the conflicting table is deleted.

This verifies that conflict checking doesn't prevent creating a saved query once
the conflicting table no longer exists.
